# 毕设宣传网页设计方案

> 论文题目：面向长期任务的 VLM-VLA 分层框架通用记忆系统研究
> 作者：王树勋　导师：郭长勇

## 设计目标

1. **主页**：3 秒抓住注意力，30 秒让人觉得"这工作做得真多"。靠数字和成果"砸"出工作量。
2. **子页面**：以讲故事的形式展开内容，按"为什么—怎么做—怎么验证—发现了什么"的叙事节奏组织。
3. **整体结构**：1 个主页 + 7 个子页面。

## 子页面

### Page 1 — 为什么需要记忆？（背景与挑战）

**故事钩子**：让机器人取杯倒水，它会忘记自己刚开过柜门。

- VLA 模型时间线（图 1-1）：从 RT-1 到 π₀.₅ 的发展脉络
- 已有方案三类记忆对比（图 1-2）：上下文扩展 / 外部存储 / 端到端嵌入
- 留给本文的问题：通用、轻量、解耦 → 引出下一页

### Page 2 — 方法核心：三步走（架构与设计）

**故事钩子**：把"记忆"拆成一个三阶段流水线，每一阶段只做一件事。

- 整体架构图（图 3-1）放最显眼位置
- 三个图解依次展开：
  - **① 提取** — 可学习记忆词元如何拼到 VLM 末端读取信息
  - **② 聚合** — 块级因果注意力如何融合滑动窗口内的历史（图 3-2）
  - **③ 注入** — 门控交叉注意力如何把记忆注入动作专家（图 3-3）
- 训练对采样的特殊要求：连续回合采样 vs 固定窗口（图 3-4）
- 强调一句话：VLM 骨干一行代码不动

### Page 3 — 不止一种方案：4 种记忆系统的对比实现（**工作量页 1**）

**故事钩子**：为了找到最好的方案，我们把四条路全走了一遍。

- 表 3-1 四种方案对比放顶部
- 每种方案一个小卡片：设计动机 / 实现要点 / 何时不适合
  - **Cache**：暴力但有注意力捷径
  - **Comp**：压缩 KV 但仍走骨干
  - **Norm**：仿射调制但缺空间选择性
  - **GCA（本文）**：解耦注入
- 关键叙事：不是只做一个方案，而是**实现并训练了四个完整模型再对比**

### Page 5 — 实验结果（仿真 + 真机）

**故事钩子**：在三个平台上分别问不同的问题。

#### 训练过程展示

**故事钩子**：6 次完整训练，曲线告诉你"这套系统是真训出来的"。

- **4 种方法的 Loss 收敛曲线对比**（同一坐标系，4 条曲线）：可视化 Cache / Comp / Norm / GCA 的训练动态，突出 GCA 收敛更稳、最终损失更低
- **三个评测平台的训练曲线**：SimplerEnv（30k 步全参数微调）、RMBench（30k 步 LoRA）、ACONE（30k 步全参数微调），分别独立展示
- **关键超参数曲线**：分组学习率（基座 5e-5 / 记忆模块 1e-4 / 门控 5e-6）的训练曲线对照，体现"门控低学习率慢热"的设计意图
- **训练资源说明**：8×H800 训练，单次约 30k 步，把"工程量"以图表形式落地

> 推荐用 Plotly / ECharts 可交互曲线图：鼠标悬停显示步数、loss、学习率；图例可单独开关每条曲线。

#### 测试结果

按任务难度递增组织：

- **SimplerEnv（短程）**
  - 表 4-3 完整对比 + 图 4-1 任务图
  - 讲点：超越 11 种基线，包括 GR00T N1.5 和 π₀-FAST
  - **视频展示**：四任务执行 rollout 视频，建议 2×2 网格自动循环播放（勺子到毛巾 / 胡萝卜到盘子 / 绿块叠黄块 / 茄子入黄篮）；可附"基座 vs 本文方法"并排对比版本，让差异肉眼可见
- **遮挡场景案例**（图 4-11）
  - 一个具体的成功例子，讲为什么记忆有用
  - **视频展示**：单任务遮挡帧逐帧慢放，对照"无记忆模型偏移目标"与"GCA 仍准确放置"两段轨迹

### Page 6 — 深入分析：为什么 GCA 赢了？

**故事钩子**：Comp 在平均分上接近 GCA，但它有一个根本性缺陷。

体现思考深度的地方，不只是报数字：

- **融合方式之差（4.3.2）**：Norm 在某任务高、某任务低，因为缺空间选择性
- **注意力捷径（4.3.3）**：Comp 让历史信息穿透 VLM 所有层，长程任务的隐患
- 配示意图：用一张图把"骨干受影响 vs 不受影响"讲清楚
- 4.3.1 聚合模块消融、4.3.4 推理效率：标"待补充"或后续填

## 落地建议

1. **主页"工作量"叙事**靠三组数字：方法数（4 种）、平台数（3 个）、数据规模（37 / 34205）。这三组数字远比"我们做了很多实验"有说服力。
2. **Page 3、Page 4 是亮点页**——很多毕设网站只有"方法+结果"，把基线实现单独拉出来 + 自研工具单独拉出来，立刻不一样。
3. **每个子页结尾放一张"下一页指引"卡片**，引导读者顺着故事往下走，而不是看完一页就关。
4. **统一视觉锚点**：图 3-1 整体架构图反复出现（首页一次、Page 2 大图、Page 5/6 小图高亮当前讨论部分），让外行也能跟上。
